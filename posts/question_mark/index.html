Question mark usage when access to an object’s property

Sometimes when try to access a particular property in an object but we are not sure whether the object has the property. Under this circumstance, we can use question mark to avoid errors. Let’s say an object person:

const person = {
name: "David",
age: 30,
gender: "Male"
};

But what happens if we try to access a “hasGirlfriend” attribute in this object?

console.log(person.hasGirlfriend);

This will cause an “undefined” result.

Then what if the object is null? In React, we use userState hook to define an default value of an variable, but if we treat the variable as null, then when we want to treat this variable as an object and try to access an attribute in this object, an typeError will occur. That’s because we cannot get an attribute from a “null” like what we do in an object.

To fix this issue, we can use a question mark to prevent this error:

const person = {
name: "David",
age: 30,
gender: "Male"
};
console.log(person.age); // output: 30

const person1 = null
console.log(person1.age); // output: TypeError
console.log(person1?.age); // output: undefined

As we can see, if we add an question mark before we try to access an attribute from an object(probably a null), the complier will first try to find the object(if exists) and does it job, if the object is a null, it will still output an undefined result instead of an error.
